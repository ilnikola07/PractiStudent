# Система "Абитуриент"

Десктопное приложение на C# для управления данными абитуриентов с разграничением прав доступа и работой с Microsoft Access.

---

## Описание

Система предназначена для учёта абитуриентов образовательного учреждения. Поддерживает два уровня доступа: администратор (полный доступ) и гость (только просмотр и отчёты).

### Технологии
*   **Язык и платформа:** C#, .NET 8, Windows Forms
*   **СУБД:** Microsoft Access (OLE DB)
*   **Архитектура:** Трёхуровневая архитектура (3-Tier Architecture)

---

## Функциональные возможности

### Общие функции
*   Авторизация и регистрация пользователей
*   Поиск, фильтрация и сортировка данных
*   Формирование отчётов в форматах Word и TXT

### Функции администратора
*   Добавление, редактирование и удаление записей
*   Каскадное удаление связанных записей
*   Просмотр таблицы пользователей

### Ограничения системы
*   Максимум 3 администратора в системе
*   Поле **"Роль"** недоступно для редактирования
*   Запрет на удаление последнего администратора

---

## Требования

### Системные требования
*   Windows 10 / 11
*   .NET 8 Runtime
*   Microsoft Access Database Engine 2016

## Установка

### 1. Microsoft Access Database Engine
Скачайте установщик с официального сайта Microsoft:
*   `AccessDatabaseEngine.exe` (32-bit)
*   `AccessDatabaseEngine_X64.exe` (64-bit)

> **Важно:** Разрядность движка (32/64-bit) должна строго совпадать с разрядностью скомпилированного приложения.

### 2. .NET 8 Runtime
Загрузите и установите актуальную версию с сайта [://microsoft.com](https://://microsoft.com/).

### 3. Приложение
Скопируйте папку с приложением в удобное место на диске. Убедитесь, что внутри папки присутствуют:
*   `StudentSource.exe` (исполняемый файл)
*   `Abiturient.accdb` (файл базы данных)
*   Все необходимые DLL-файлы

---

## Запуск

### Из готовой сборки
1. Запустите файл `StudentSource.exe`
2. Нажмите кнопку **"Подключиться к БД"**
3. Выберите файл базы данных `Abiturient.accdb`

### Из исходного кода
1. Откройте файл решения `PractiStudent.sln` в **Visual Studio 2022**
2. Установите проект `StudentSource` как стартовый проект
3. Нажмите клавишу `F5` для запуска отладки

## Руководство пользователя

### Вход в систему
Учётная запись администратора по умолчанию:
*   **Логин:** `admin`
*   **Пароль:** `admin123`

*Примечание: При самом первом подключении к чистой базе данных администратор создаётся автоматически.*

---

### Роль: Гость

**Доступные действия:**
*   Просмотр таблиц (кроме таблицы "Пользователи")
*   Поиск и фильтрация данных
*   Сортировка записей
*   Формирование отчётов

**Ограничения:**
*   Нельзя добавлять, редактировать или удалять записи

---

### Роль: Администратор

**Доступные действия:**
*   Все функции гостя
*   Добавление новых записей
*   Редактирование существующих записей
*   Удаление записей (с каскадным удалением для справочников)
*   Просмотр таблицы пользователей

**Важные правила:**
*   Нельзя удалить самого себя
*   Нельзя удалить последнего администратора в системе
*   Нельзя вручную добавлять пользователей через таблицы (только через форму регистрации)

---

### Каскадное удаление
При удалении записей из справочников автоматически удаляются связанные данные по следующей цепочке:
*   **Факультет** → все специальности → специализации → абитуриенты
*   **Специальность** → все специализации → абитуриенты
*   **Специализация** → все абитуриенты этой специализации
*   **Учебное заведение** → все абитуриенты этого заведения
