Система "Абитуриент"
Десктопное приложение на C# для управления данными абитуриентов с разграничением прав доступа и работой с Microsoft Access.
Описание
Система предназначена для учёта абитуриентов образовательного учреждения. Поддерживает два уровня доступа: администратор (полный доступ) и гость (только просмотр и отчёты).
Технологии:
C#, .NET 8, Windows Forms
Microsoft Access (OLE DB)
Трёхуровневая архитектура
Функциональные возможности
Общие функции:
Авторизация и регистрация
Поиск, фильтрация, сортировка данных
Формирование отчётов (Word, TXT)
Функции администратора:
Добавление, редактирование, удаление записей
Каскадное удаление связанных записей
Просмотр таблицы пользователей
Ограничения:
Максимум 3 администратора
Поле "Роль" недоступно для редактирования
Запрет на удаление последнего администратора
Требования
Системные:
Windows 10/11
.NET 8 Runtime
Microsoft Access Database Engine 2016
Установка
1. Microsoft Access Database Engine
Скачайте с официального сайта Microsoft:
AccessDatabaseEngine.exe (32-bit)
AccessDatabaseEngine_X64.exe (64-bit)
Важно: разрядность движка должна совпадать с разрядностью приложения.
2. .NET 8 Runtime
Загрузите с dotnet.microsoft.com
3. Приложение
Скопируйте папку с приложением в удобное место. Убедитесь, что присутствуют:
StudentSource.exe
Abiturient.accdb (база данных)
Все DLL-файлы
Запуск
Из готовой сборки:
Запустите StudentSource.exe
Нажмите "Подключиться к БД"
Выберите файл Abiturient.accdb
Из исходного кода:
Откройте PractiStudent.sln в Visual Studio 2022
Установите StudentSource как стартовый проект
Нажмите F5
Руководство пользователя
Вход в систему
Учётная запись администратора по умолчанию:
Логин: admin
Пароль: admin123
При первом подключении администратор создаётся автоматически.
Гость
Доступные действия:
Просмотр таблиц (кроме "Пользователи")
Поиск и фильтрация данных
Сортировка записей
Формирование отчётов
Ограничения:
Нельзя добавлять, редактировать или удалять записи
Администратор
Доступные действия:
Все функции гостя
Добавление новых записей
Редактирование существующих
Удаление записей (с каскадом для справочников)
Просмотр таблицы пользователей
Важные правила:
Нельзя удалить себя
Нельзя удалить последнего администратора
Нельзя добавлять пользователей (только через регистрацию)
Каскадное удаление:
При удалении факультета/специальности/специализации автоматически удаляются все связанные записи.
